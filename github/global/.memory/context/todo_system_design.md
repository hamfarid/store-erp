# تصميم نظام TODO

## الهدف
نظام متكامل لتتبع المهام بدقة مع ثلاثة ملفات:
1. `TODO.md` - الخطة الكاملة (لا يُحذف منها أبداً)
2. `COMPLETE_TASKS.md` - المهام المكتملة
3. `INCOMPLETE_TASKS.md` - المهام غير المكتملة

---

## الملفات الثلاثة

### 1. TODO.md
**الغرض:** السجل الدائم لجميع المهام

**القواعد:**
- ✅ يتم إنشاؤه في Phase 1
- ✅ يحتوي على الخطة الكاملة بالتفصيل الدقيق
- ✅ لا يُحذف منه أي شيء أبداً
- ✅ يتم وضع [x] بجانب المهمة المكتملة
- ✅ يتم وضع [ ] بجانب المهمة غير المكتملة
- ✅ يُحدث باستمرار

**الهيكل:**
```markdown
# TODO List

## Phase 1: Initialize
- [x] مهمة مكتملة
- [ ] مهمة غير مكتملة
- [ ] مهمة أخرى

## Phase 2: Plan
- [ ] مهمة 1
- [ ] مهمة 2
```

---

### 2. COMPLETE_TASKS.md
**الغرض:** قائمة المهام المكتملة فقط

**القواعد:**
- ✅ يتم تحديثه تلقائياً
- ✅ يحتوي على المهام المكتملة فقط
- ✅ يتم نقل المهمة من INCOMPLETE إلى COMPLETE عند الإنجاز
- ✅ يحتوي على تاريخ الإنجاز

**الهيكل:**
```markdown
# Completed Tasks

## 2025-11-15
- [x] مهمة 1 - تم في 13:00
- [x] مهمة 2 - تم في 14:30
```

---

### 3. INCOMPLETE_TASKS.md
**الغرض:** قائمة المهام غير المكتملة فقط

**القواعد:**
- ✅ يتم تحديثه تلقائياً
- ✅ يحتوي على المهام غير المكتملة فقط
- ✅ يتم حذف المهمة منه عند الإنجاز
- ✅ يتم إضافة مهام جديدة عند اكتشافها

**الهيكل:**
```markdown
# Incomplete Tasks

## High Priority
- [ ] مهمة حرجة 1
- [ ] مهمة حرجة 2

## Medium Priority
- [ ] مهمة متوسطة 1

## Low Priority
- [ ] مهمة منخفضة 1
```

---

## سير العمل

### عند بدء المشروع (Phase 1)
1. إنشاء `docs/TODO.md`
2. كتابة الخطة الكاملة بالتفصيل
3. إنشاء `docs/INCOMPLETE_TASKS.md` (نسخ من TODO)
4. إنشاء `docs/COMPLETE_TASKS.md` (فارغ)

### أثناء العمل
1. عند إنجاز مهمة:
   - وضع [x] في `TODO.md`
   - نقل المهمة من `INCOMPLETE_TASKS.md` إلى `COMPLETE_TASKS.md`
   - إضافة تاريخ ووقت الإنجاز

2. عند اكتشاف مهمة جديدة:
   - إضافتها في `TODO.md`
   - إضافتها في `INCOMPLETE_TASKS.md`

3. في نهاية كل مرحلة:
   - مراجعة `TODO.md`
   - التأكد من تطابق الملفات الثلاثة

---

## الموقع

```
docs/
├── TODO.md                 # السجل الدائم
├── COMPLETE_TASKS.md       # المهام المكتملة
└── INCOMPLETE_TASKS.md     # المهام غير المكتملة
```

---

## الفوائد

1. **تتبع دقيق:** معرفة ما تم وما لم يتم بدقة
2. **سجل دائم:** TODO.md يحتفظ بكل شيء
3. **رؤية سريعة:** INCOMPLETE_TASKS.md يظهر ما تبقى فقط
4. **تاريخ الإنجاز:** COMPLETE_TASKS.md يوثق متى تم كل شيء
5. **منع النسيان:** لا يتم حذف أي شيء من TODO.md

---

## الإلزام

**لا يمكن إكمال Phase 1 بدون:**
- ✅ إنشاء `docs/TODO.md`
- ✅ إنشاء `docs/INCOMPLETE_TASKS.md`
- ✅ إنشاء `docs/COMPLETE_TASKS.md`

**لا يمكن إكمال أي مرحلة بدون:**
- ✅ تحديث `TODO.md`
- ✅ تحديث `COMPLETE_TASKS.md`
- ✅ تحديث `INCOMPLETE_TASKS.md`

**قبل التسليم النهائي:**
- ✅ جميع المهام في `TODO.md` عليها [x]
- ✅ `INCOMPLETE_TASKS.md` فارغ
- ✅ `COMPLETE_TASKS.md` يحتوي على جميع المهام
