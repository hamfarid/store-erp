# دليل استخدام النظام العالمي الاحترافي v35.0 (نسخة التفرد)

مرحباً بك في **Global Professional System v35.0**. هذا النظام ليس مجرد مجموعة من السكربتات، بل هو **محرك ذكاء اصطناعي هجين (Hybrid Engine)** يدمج بين بروتوكول **Speckit** الصارم وبين **القوانين العالمية (Global Rules)** الخاصة بك، لضمان أداء فائق الجودة وخالٍ من الهلوسة.

---

## 1. الفلسفة الجديدة: لماذا v35.0؟

تم تصميم هذه النسخة لتحقيق "الأداء الأقصى" (Maximum Performance) من خلال أربعة مبادئ صارمة:

1.  **المحرك الهجين (Hybrid Engine):** لا نقوم بمحاكاة Speckit، بل ننفذه فعلياً مع حقن قوانينك (مثل التفكير البصري، والتوثيق الإلزامي) داخل كل خطوة.
2.  **الكود هو الذاكرة (Code is Memory):** النظام لا ينسى. أي كود يُكتب يتم فهرسته فوراً في `code_structure.json`.
3.  **الحارس الصارم (The Overlord):** أداة `lifecycle.py` ترفض أي مخرجات "كسولة". لا يوجد مخطط Mermaid؟ يتم رفض المواصفات. لا يوجد Docstrings؟ يتم رفض الكود.
4.  **التفكير الميتا-معرفي (Meta-Cognition):** الذكاء الاصطناعي يُجبر على محاكاة نقاش بين (المهندس المعماري، البنّاء، والناقد) قبل اتخاذ أي قرار.

---

## 2. هيكلية النظام

يحتوي المجلد `global/` على الأدوات التالية:

*   **`tools/lifecycle.py` (المايسترو):** القائد الذي يدير العمليات بالتسلسل الصارم.
*   **`tools/mission_control.py` (لوحة التحكم):** شاشة تعرض حالة المشروع، الذاكرة، والمهام في الوقت الفعلي.
*   **`tools/code_indexer.py` (الفهرس):** الأداة التي تقرأ الكود وتحوله إلى ذاكرة للنظام.
*   **`prompts/speckit/*.md`:** التعليمات الصارمة لكل مرحلة (Specify, Plan, Implement, etc.).
*   **`prompts/GLOBAL_PROFESSIONAL_CORE_PROMPT_v35.0.md`:** البرومبت الرئيسي الذي يجب استخدامه مع الذكاء الاصطناعي.

---

## 3. كيفية البدء (دليل خطوة بخطوة)

### الخطوة الأولى: التحضير
تأكد من وجود مجلد `global` في جذر مشروعك.

### الخطوة الثانية: تشغيل المايسترو (The Maestro)
هذا الأمر يبدأ دورة حياة المشروع. استبدل القيم بما يناسبك:

```bash
python3 global/tools/lifecycle.py [اسم_المشروع] "[وصف_المهمة]"
```

*مثال:*
```bash
python3 global/tools/lifecycle.py MyProject "Build a secure API for a task manager"
```

**ماذا سيحدث؟**
1.  **Reality Check:** النظام يمسح المجلد الحالي ويفهرس كل شيء.
2.  **Constitution:** يكتب دستور المشروع.
3.  **Specify:** يطلب منك المواصفات (ويجبرك على رسم مخططات).
4.  **Plan:** يحول المواصفات إلى خطة تقنية.
5.  **Implement:** يبدأ التنفيذ (كتابة كود + اختبار + توثيق).

### الخطوة الثالثة: المراقبة (Mission Control)
في نافذة تيرمينال أخرى، يمكنك مراقبة ما يحدث:

```bash
python3 global/tools/mission_control.py
```
ستظهر لك شاشة تعرض عدد الملفات المفهرسة، نسبة إنجاز المهام، وحالة النظام.

---

## 4. القوانين الصارمة (The Iron Rules)

عندما يعمل النظام، فإنه يلتزم بالقوانين التالية (ولا يمكنه كسرها):

1.  **قانون الرسم (Visual Thinking):** أي مواصفات (`/speckit.specify`) لا تحتوي على مخطط Mermaid تعتبر **مرفوضة**.
2.  **قانون التوثيق (Docstring Enforcement):** أي دالة (Function) أو فئة (Class) تُكتب بدون شرح (Docstring) تعتبر **خطأ** ويجب إصلاحها فوراً.
3.  **قانون الذاكرة (Indexer):** بعد كل عملية تعديل على الكود، يتم تشغيل `code_indexer.py` تلقائياً لتحديث الذاكرة.

---

## 5. ماذا تفعل إذا توقف النظام؟

إذا توقف النظام أو طلب منك تدخلاً:
1.  اقرأ الرسالة جيداً. غالباً ما يطلب منك "توضيح" (`/speckit.clarify`) لغموض معين.
2.  أعطه المعلومات المطلوبة.
3.  سيقوم النظام بتحديث خطته والمتابعة.
