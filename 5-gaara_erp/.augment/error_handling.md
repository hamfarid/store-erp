## Purpose
Handle errors systematically. Learn from failures. Improve continuously.

## Core Principle
**Errors are learning opportunities. Handle them systematically.**

## Error Resolution Protocol

### Attempt 1: Internal Fix
- Try to fix based on knowledge
- Apply common solutions
- Check obvious issues
- Quick fix attempt

### Attempt 2: Review Rules
- Read relevant rules file
- Check guidelines
- Apply documented solutions
- Follow best practices

### Attempt 3: Deep Analysis
- Analyze root cause
- Check dependencies
- Review context
- Think creatively
- Try alternative approaches

### Attempt 4+: Search Internet
- Search error message
- Read Stack Overflow
- Check GitHub issues
- Find similar problems
- Learn from solutions
- Apply and adapt

## Error Documentation

### Record Every Error
```
Error: [description]
Cause: [root cause]
Solution: [what fixed it]
Prevention: [how to avoid]
Learned: [lessons learned]
```

## Remember
**3 attempts internally, then search externally.**
**Every error is a lesson. Document and learn.**

Handle errors systematically. Always.
